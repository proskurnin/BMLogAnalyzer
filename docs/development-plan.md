# План разработки

Этот документ фиксирует актуальное состояние после реализации задач по анализу НБС из архивов `13-07-2026.zip` и `kzp_logs_260715.zip` и доработок блока `Состав загрузки`.

Ключевое правило сохраняется: факты из логов, гипотезы и рекомендации должны быть разделены. Нельзя делать вывод без evidence-строк с `source_file`, `line_number`, `timestamp` и `raw_line`.

## Реализовано

1. Точная корреляция NBS `Send Commands::info` и BM `Info, req/resp`:
   - извлекаются `rid`, `BmNumber`, `TmSerialNumber`, `tm serial number`, `reader id`;
   - BM Info сопоставляется только по совпавшему device id или при доказуемо единственном устройстве в окне;
   - при неоднозначности отчёт показывает статус `ambiguous` и не строит причинную цепочку;
   - в manifest добавлены интервалы `Send Commands::info -> BM Info req`, `BM Info req -> BM Info resp`, `BM Info resp -> QR`.

2. Сверка с выводами коллег по `kzp_logs_260715.zip`:
   - найдено 7 сценариев выхода НБС дольше 3 секунд;
   - проблемными классифицируются 5 сценариев;
   - среднее `MODE::VALIDATE -> первый QR` для проблемных сценариев: `81,702 сек`;
   - исключённые сценарии остаются в отчёте с причиной исключения.

3. Расширенная статистика `StopListDb` / `References_nbs_slm`:
   - считаются `count`, `min`, `max`, `average`, `median`, `p90`, `p95`;
   - выбросы выделяются относительно фактов того же сценария;
   - внешний норматив не используется;
   - max и выбросы выводятся с evidence-строками.

4. Сравнение по фактическим границам смены:
   - сценарии получают фазу `after_log_started`, `after_session_closed` или `unknown`;
   - HTML показывает сравнение по фазам только если есть фактические группы для сравнения.

5. Каталог QR-state маркеров:
   - фиксируются прямые маркеры `QR data`, пустой `QR data`, `qrData`, `settime`, `updateConfiguration`, очистка/сброс QR;
   - отчёт не пишет `QR пропал` без прямого marker evidence.

6. HTML и manifest:
   - в блок `Выход НБС в работу после открытия смены` добавлены классификация, device id, BM Info-корреляция, расширенная статистика StopListDb/References и QR-маркеры;
   - текстовый отчёт содержит факты, причины включения/исключения и evidence.

7. Блок `Состав загрузки`:
   - счётчик log-файлов в шапке синхронизирован с фактическим inventory архива;
   - размеры файлов показываются в `KB`, если значение меньше 1 MB;
   - список `Он содержит логи следующих типов` строится по фактическим log-файлам архива;
   - модальное окно по типу лога показывает список файлов этого типа, количество строк совпадает со счётчиком;
   - типы логов вынесены в единый справочник, используемый summary, manifest, section registry и HTML;
   - при полном покрытии сканером типы строятся по content/path-классификации файлов, при неполном покрытии используется archive inventory без двойного учёта;
   - в модалке по типу лога выводятся размер файла и причина классификации;
   - `Evidence` в таблице распознавания типов переименован в `Признак распознавания`;
   - таблица `Распознанные типы логов` оформлена зеброй;
   - в `Разделы отчёта и источники` вместо `Не найдены` используется колонка `Найдены` с отметками ✅ / ❌.
